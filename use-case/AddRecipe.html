Add Recipe (Overview)

Task Flow Diagram
UC Diagram

This use case details how the Chef will use the system to create a recipe.

  1. Basic Flow
    1. Select Option

      2. The use case begins when the Chef selects the option for create recipe.

    2. Display retailer list screen

      3. The system displays the retailer list screen.

    3. Select retailer

      4. The Chef chooses a retailer.

    4. Display recipe summary screen

      5. The system will display a create recipe screen.

    5. Enter recipe summary

      6. The Chef enters the recipe summary [Servings, Main Ingredient, Cuisine type, Cost limit]

    6. Add recipe details

      7. The Chef chooses to add the recipe details.

    7. Display recipe screen

      8. The system displays the recipe page.

      [This use case loops from step 1.8 (Enter ingredient) to step 1.8 (Enter ingredient) until the Chef chooses to enter cooking details]

    8. Enter ingredient

      9. The Chef enters an ingredient, quantity and unit of measure.

    9. Enter cooking details

      10. The Chef enters the cooking details [instructions, cooking time, extra tips & tricks].

    10. Select Save recipe option

      11. The Chef chooses to save the Recipe.

    11. Save recipe

      12. The system saves the recipe.

    12. Display recipe summary

      13. The system displays the recipe summary page and the use case ends.

  2. Alternative Flows
    1. Copy Shopping List

      2. If at step 1.5 (Enter recipe summary) the Shopper chooses to copy an existing shopping list, the Shopper selects a shopping list from the list of shopping lists. The system loads a new version of the chosen shopping list and the use case continues at step 1.7 (Display recipe screen).

    2. Enter calories per serving

      3. If at step 1.5 (Enter recipe summary) the Chef chooses to add ‘calories per serving’ to the recipe. The system displays the appropriate screen. The Chef enters calories per serving for the recipe and selects the save calories per serving option. The use case continues at step 1.5 (Enter recipe summary).

    3. View nutrition

      4. If at step 1.8 (Enter ingredient) the Chef chooses to view nutritional data for the ingredient.. The system displays the appropriate screen showing the nutritional data. The Chef closes the screen and the use case continues at step 1.8 (Enter ingredient).

    4. Find alternative ingredient

      5. If at step 1.8 (Enter ingredient) the Chef chooses to find alternative ingredient. The system displays the appropriate screen. The Chef chooses an alternative ingredient and chooses to save the alternative ingredient. The system saves the alternative ingredient and the use case continues at step 1.8 (Enter ingredient).

    5. Rate recipe

      6. If at step 1.12 (Display recipe summary) the Chef chooses to rate the recipe. The system displays the appropriate screen. The Chef enters a rating for the recipe and chooses to save the rating. The system saves the rating. The use case continues at step 12 (Display recipe summary).

    6. Cancel

      7. If at any time the Chef chooses to cancel the activity, the system displays a warning message asking the Chef to either continue with the cancel action or quit the cancel action. If the Chef chooses to continue with the cancel action, the use case ends. If the Chef chooses to quit the cancel action, the system closes the message and the use case continues from its current point.

    7. Database unavailable

      8. If at step 1.11 (Save recipe) the system establishes that the database is unavailable, the system displays a warning message. The Chef acknowledges the warning message and the use case ends.

  3. Key Scenario

    4. Gillian Moore just wants an app that will help her cook healthy without the need to be a Michelin Star chef or have a pantry with exotic ingredients. She just wants a simple recipe that will not have to break the bank.

  4. Preconditions

    5. The Chef must have successfully logged in, and still be logged in.

  5. Post-conditions

    6. Either the recipe has been created and all data has been saved, or no data has been modified.
