Get Food Nutritional Info

This use case details how the Shopper will use the system to scan food barcodes to determine the nutritional information about the food product.

  1. Basic Flow
    1. Select Option

      2. The use case begins when the Shopper selects the option for scanning the barcode of a food product.

      [The use case loops from step 1.4 (Display scan food barcode screen) to step 1.7 (Display nutritional data screen) until the Shopper chooses to exit.]

    2. Display Retailer List screen

      3. The system displays the retailer list screen.

    3. Select Retailer

      4. The Shopper chooses a retailer.

    4. Display scan food barcode screen

      5. The system displays the food barcode screen.

    5. Scan food product

      6. The Shopper scans food product.

    6. Retrieve nutritional data

      7. The system retrieves the nutritional data for the scanned item.

    7. Display nutritional data screen

      8. The system displays the nutritional information relating to the scanned product.

    8. Select compare similar products option

      9. The Shopper chooses to compare similar products.

    9. Display similar products comparison

      10. The system retrieves and displays similar products to the scanned food product, and the use case ends

  2. Alternative Flows
    1. Sort similar products

      2. If at step 1.8 (Display similar products comparison) the Shopper chooses sort the list by either by price or nutritional benefit, the system displays the sorted list. The use case continues at step 1.8 (Display similar products comparison).

    2. Search for food product

      3. If at step 1.4 (Scan food product) the Shopper chooses to search for the food product. The system displays the appropriate screen. The user enters the food product name and brand. The use case continues at step 1.5 (Retrieve nutritional data).

    3. Display recipes

      4. If at step 1.6 (Display nutritional data screen) the Shopper chooses to display recipes containing the scanned product. The system displays a list of recipes which contain the scanned product. The use case continues at step 1.4 (Scan food product).

    4. View Tutorial

      5. If at step 1.3 (Select Retailer) the Shopper chooses to view the tutorial, the system displays the appropriate screen. The Shopper chooses to close the tutorial. The system closes the tutorial screen, and the use case continues at step 1.3 (Select Retailer).

    5. Show Alternative Foods

      6. If at step 1.7 (Select compare similar products) the Shopper chooses to view information on alternative foods, the system displays the appropriate screen. The Shopper chooses to scan the next barcode and the use case continues at step 1.4 (Scan Food Product).

    6. Cancel

      7. If at any time the Shopper chooses to cancel the activity, the system displays a warning message asking the Dieter to either continue with the cancel action or quit the cancel action. If the Dieter chooses to continue with the cancel action, the use case ends. If the Dieter chooses to quit the cancel action, the system closes the message and the use case continues from its current point.

    7. Database unavailable

      8. If at step 1.5 (Retrieve nutritional data) or step 1.8 (Display similar products comparison) the system establishes that the database is unavailable, the system displays a warning message. The Shopper acknowledges the warning message and the use case ends.

  3. Key Scenario

    4. Lorcan Duignan never really cared about healthy food but due to a type 2 diabetes condition needs to watch what he is eating and found the barcode scanner to be simple to use. All he had to do was swipe it on and the icon for the scanner was ready to go and then it just swiped all the details. It even allowed him to know if there was too much fat, salt or sugar content in that particular item. He was able to make an informed health decision based on his own profile and also compare and contrast food prices based on nutritional value!

  4. Preconditions

    5. The Shopper must have successfully logged in, and still be logged in.

  5. Post-conditions

    6. None.
